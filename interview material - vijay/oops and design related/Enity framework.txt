1) What is enity framework?
  - In .net applications, ADO.NET is an old & powerful method for accessing databases and requires a lot of coding
  - Microsoft therefor introduced Entity Framework to automate database technique.
  - It basically supports ORM (Object Relational Mapper) basically it maps tables & columns in Database with various objects in application.

2) What is the ORM model in C#?
  - ORM is a technique that maps object-oriented models to relational database models
  - Object-relational mapping (ORM) is a technique that creates a layer between the language and the database, 
    helping programmers work with data without the OOP paradigm.

3) What other O/RMs you can use with .NET based applications?
  - Entity Framework 6.x
  - Entity Framework Core
  - Dapper

4) What is Micro O/RMs?
  - A micro ORM is designed to focus on the most important task of working with database tables instead of
    of creating, modyfing the database schema, tracking changes etc. 
  - EF 6.x & EF Core both are O/RM's since they provide full set of features

5) What is Dapper?
  - Dapper is simple / micro ORM for the .NET world.
  - Dapper was created by StackOverflow team to address their issues and open source it.
  - It's a NuGet library that can be added to any .NET project for database operations.

6) Why Dapper?
  - Dapper is the second-fastest ORM
  - Perform CRUD operations directly using the IDBConnection object.

7) What is SQL injection attack?
  - A SQL injection attack is an attack mechanism used by hackers to steal sensitive information from the database of an organization.
  - It is the application layer (means front-end) attack which takes benefit of inappropriate coding of our applications 
    that allows a hacker to insert SQL commands into your code that is using SQL statement.

8) How to handle SQL injection attacks in Entity Framework?
  - Entity Framework is injection safe since it always generates parameterized SQL commands 
    which help to protect our database against SQL Injection.

9) What are various approaches to domain modeling in Entity Framework?
   - Code First
   - Database first
   - Model First

10) Code First?
   - It allows us to design C# classes and then create database from those classes.
   - These classes are called POCO classes.

11) Advantages of Code First
   - It provided full control over code since there is no auto generated code which is hard to modify.
   - In this approach, your code defines only database mappings and EF will handle creation of database with its relations
   - Manual changes in database schema is not preferable as your code defines database.

12) Database first?
   - It allows us to create model from existing databse.
   - This approach reduce the amount of code to write since it automatically generates code.
   - But its limits us to work with the structure of the generated code.

13) Advantage of Database First
   - Very popular if you have databse designed by DBAs, developed seperately or if you have existing Database
   - EDM wizard generates entities, relationships, and inheritance hierarchies for you.
   - To add extra features in POCO entities, you must either modify T4 template or create partial classes.
   - Manual changes in Database possible because the defines domain model. You can always update model from database.

14) Model First?
   - It allows us to create model's entities, relationships and inheritance hierachies on the design surface of emmpty model (.edmx file)
     by using entity designer and then create database from it
   - This approach is adopted by the architect & solution lead developers.

15) Advantage of Model First
   - it is good if you like to visualize the structure of data in the application or you dont like write code or sql since it will auto generate for you
   - In this you dont have any control over centities & database, thats why it is rarely used for small easy projects.
   - To add extra feature in POCO entities, you must either modify T4 template or create partial class
   - Manual changes in Ddatabase is not possible because your code defines database.

16) What are POCO classes?
   - POCO entities are existing domain objects within your application thet you use with EF.

17) What is the proxy object?
   - This is object which is created from POCO Class or entities generated by EF to support change tracking & lazy loading
  
18) rules for creating a proxy class object.
   - The class must be public and not sealed.
   - Each property should be marked as virtual
   - Each property should have public getter or setter.
   - Any collection navigation property should be types as ICollection<T>

     - Added => This has to set explicitly when you add any new record.
     - Modified => This will set by Context if any changes are made to entity.
     - Deleted => This has to set explicitly when we delete any record.
