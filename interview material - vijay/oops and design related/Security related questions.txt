1) Best ways to implementing security?
  - Implement Authentication & AUthorization
  - Use HTTPS
  - Validate all incoming inputs (Like prevent SQL Injection, XSS Attack, command injection)
  - Prevent XSS (Cross Site Scriptting)
  - Prevent CSRF (Cross Site Request Forgery)
  - Implement encryption of sensitive information
  - Perform Security Testing
  - Add Security Headers in request for example: CSP Header to prevent XSS

2) How to prevent SQL Injection?
  - Use parameterized queries & prepared statements
  - do not constructing queries using string concatenation or interpolation.

3) How to prevent token stealing?
  - Use HTTPS
  - Keep token expiration
  - Token Encryption 
  - Secure Token Storage (dont store token in local storage or cookies, Use HTTP only cookies for storing tokens)
  - Token Binding (Bind token specific client device or browser)
  - Same Site Cookie attribute

4) Oauth 2?
  - Roles 
    - Resource Owner : Its basically user who owns the resource and grants access to it.
    - Client Application : This is application (website, mobile app) requesting access to the user's resources
    - Resource Server: This is the server which hosting protected resources (e.g user's data)
    - Authentication Server : This server authenticates the user and issues the tokens.

5) How claim based authorization works in openid connect with example?
   - User initiates authentication by accessing a client application.
   - The client application redirects the user to the Identity Provider (IDP) for authentication.
   - The IDP authenticates the user (e.g., using username/password, social login, etc.).
   - Upon successful authentication, the IDP generates an ID Token.
   - The ID Token contains claims about the authenticated user (e.g., name, email, roles).
   - The ID Token is securely transmitted back to the client application.
   - The client application receives the ID Token and extracts the claims.
   - The claims are used for authorization decisions within the client application.

6) What is role of Identity provider?
  - IDP is responsible for managing user identities and their associated attributes, including roles. 
  - This information is then used during the authentication process to generate tokens with appropriate claims
    allowing client applications to make access control decisions based on user roles.

7) How server to server call sends token?
  - By using Grant Type client credentials

8) Grant Type: 
  - Authorization Code Grant
    - For Server Side Application
    - Redirects user for authentication, exchanges code for tokens.
    - This is more secure because it exchange the sensitive information server to server.

  - Implicit Flow:
    - For Javascript based application example Single Page Application
    - Obtains tokens directly from authorization endpoint
    - Here token is exposed

  - Client Credentials
    - For Server to server communiation
