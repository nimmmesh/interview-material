1) What is microservices?
   - Miscoroservice is a small unit that has only one responsibility or single logic which solves specific problem.
   - Microservices is small & independent services that work together to build highly automated, independent & evolving software
   
2) Advantages of microservices?
   - Technology diversity
   - Independent deployment
   - Independent development
   - Provide improved scalability
   - Better fault isolation means even though one service fails it will not impact other services.
   - Zero downtime upgrade
   - We can scale them in seconds without compromising integrity of application.
   - Services can be hosted on different servers & different datacenters.
   - Support CI/CD.
  
   

3) Monolithic vs Microservices?
   - Monolithic : 
     - Single Unit. Everything we writing in single layer
	 - The service startup takes time as it is slow in Monolithic Architecture
	 - 	It is primarily a tightly coupled architecture.
	 - In Monolithic Architecture, any changes in the data model affect the entire database.
	 - It mainly focuses on the whole project.
   
   - Microservices :
     - Decompose application into smaller units.
	 - The service startup is fast in Microservices.
	 - It is a loosely coupled architecture.
	 - In Microservices, if you make changes in a single data model, it does not affect others.
	 - It mainly focuses on products, not projects.
  
   
4) What do you understand by Monolithic Architecture?
   - Monolithic architecture is like a big container that contains all the software components of an application. 
   - These applications are clubbed inside a single package within the application.

5) What are the disadvantages of monolithic architecture?
   - Slower development speed 
     A large, monolithic application makes development more complex and slower. 
   - Scalability : You can’t scale individual components.
   - Blocks continuous development
   - Barrier to technology adoption 
   - Deployment : 
     A small change to a monolithic application requires the redeployment of the entire monolith.
   - Unreliable : if any module goes down then entire application will goes done 
   - Unflexible: if we develop the application in one technogy & if you want to adopt new technology for development
     it would be difficult to incorporate this.



   
  
   
5) What are the biggest challenges in Microservice deployment?
   - Microservices require a heavy investment.
   - It requires a heavy infrastructure setup also.
   - We need excessive planning for managing operations overhead.
   - It cost a lot in staff selection and maintenance.
   - The components in microservices always rely on each other, so; it requires communication between them in the application.
   - There are also a lot of challenges in deployment.
   - Testing and Debugging are very challenging.
   - It requires complete component automation and application maintenance.
   
6) In which cases microservice architecture is best suited?
   - The microservice architecture is best suited for all tech devices such as desktop, web, mobile devices, Smart TVs, Wearable devices, etc
   
7) Which are some famous companies that are using Microservice architecture?
   - Twitter, Netflix, Amazon
   
8) What do you understand by RESTful?
   - The RESTful web service is an architectural style that helps computer systems to communicate over the internet.
