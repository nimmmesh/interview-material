1) How to improve application performence?
    - Use stringbuilder instead of string
    - Use caching
    - Avoid boxing and unboxing
    - Dont write inline queries
    - Use Stored procedure
    - Effectively use CSS and Script files (Bundling & minification)
    - Validate using JavaScript
    - Avoid unnecessary roundtrips
    - Clear the Garbage Collection
    - Implement Dynamic Paging
    - Use Threads eg on one click send 5000 emails.
    - Use Parallel Programming

2) How to improve front end performence?
    - Minification & compression
      Minimize and compress HTML, CSS, and JavaScript files to reduce page load times.
    - Caching
    - Lazy loading
    - Use CDN
      - Use CDN to deliver static assets from servers geographically closer to user
    - Reducing HTTP Requests
    - Implement paging
    - Use OnPush Change detection stratergy.

3) How to improve Database performence?
    - Indexing
    - Query Optimization : 
      - Reduce unnecessary joins
      - Select required columns
    - Avoid subqueries
    - Use Exists instead of In.
    - Keep clustered index short
    - Avoid cusrsor
    - Caching
    - Normalization 
    - Database Sharding : 
      Distribute data across multiple databases or shards to distribute the load.
    - Partitioning
      Partition large tables to improve query performance.

4) Can you explain the steps you would take to identify performance bottlenecks in a web application?
    - Profiling & Monitoring tools 
      Such as New Relic tools.
    - Load Testing
    - Database Optimization
    - Caching
    - Concurrency
    - Use CDN to serve static assets and reduce server load

5) Downtime handling approach?
   - Immediate Assessment: Quickly identifying the root cause.
   - Effective Communication: Notifying relevant team members promptly.
   - Isolation and Troubleshooting: Isolating affected components and employing troubleshooting techniques.
   - Temporary Fix: Implementing a quick solution to restore functionality.
   - Thorough Documentation: Keeping detailed records of actions taken.
   - Post-Incident Analysis: Identifying root cause and implementing preventative measures.
   - Transparent Communication: Keeping stakeholders informed of resolutions and preventive actions.
   - Continuous Learning: Committing to ongoing skill development for future challenges."

6) How to check if which part of stored procesure taking so much time?
   - Query execution plan
   - In SQL Server Management Studio (SSMS), you can generate an execution plan by pressing CTRL + M.
   - Query Profiler, SQL Server Profiler is a tool to capture & analyze events in SQL Server
     You can create trace session that captures the execution of SP and see which parts are consuming the 
     most time
