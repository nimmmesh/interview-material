- What is Normalization?
  - Its database design technique to avoid redundant data in tables

- What is Denormalization?
  - Its exactly opposite of Normalization where we allow redundant data to improve search performence.

- What is OLTP & OLAP?
  - OLTP Stands for Online Transaction Process where we store normalize data in this database
  - This is useful for any CRUD operation
  - OLAP stands for Online Analytical Process where we store DeNormalize data in this database.
  - This is useful to improve search performence
  - This is usefult for Reporting & Creating Dashboards purpose.
 

- What is 1st NF?
  - Our table should have atomic values, it should not have repeating groups
    For Example:
    - Name Columns in Emplyee tables : should not have First Name, Middle Name & Last name in single columns
      if we do this then its difficult to do update, select. To avoid this we should create seperate columns
      for first name, middle name & last name

- What is 2nd NF?
  - It should satisfied all rules of 1st NF.
  - Also any Non-Columns should be fully dependent on the primary key

- What is 3rd NF?
  - It should satisfied all rules of 1st & 2nd NF
  - Also shoul not be any non-key columns should not be dependant on any non key columns.
  - Product & Cost columns example.
  

1) NULLIF
    1) NULLIF returns the first expression if the two expressions are not equal. 
    2) If the expressions are equal, NULLIF returns a null value of the type of the first expression.

2) Stored procedure and Function
    Stored procedure
    1) Stored procedure are precompiled objects which are compiled first time and its compiled version
       are saved which executes whenever it is called.
    2) may or may not return value
    3) Procedure can have input/output parameter
    4) Procedure cannot called from function
    5) Procedure allow Select as well as DML statements
    6) Allow try catch
    7) Allow transaction management
    8) Cannot call procedure in select statement

    Function
    1) Function compiled and executed everytime when it is called
    2) Function must return value.
    3) Function have only input parameter
    4) Function can called from stored procedure
    5) Function allow only select statements	
    6) Not allow try catch
    7) Not allow transaction management.
    8) Can call function in select statement


3) Indexes?
    - Indexes in SQL Server are similar to the indexes in books. They help SQL Server retrieve the data quicker. Indexes are of two types. Clustered indexes and non-clustered indexes. 
    - Clustered indexes
      When you create a clustered index on a table, all the rows in the table are stored in the order of the clustered index key. So, there can be only one clustered index per table. 
    - NonClustered indexes
      Non-clustered indexes have their own storage separate from the table data storage.

4) Difference between clustered and non-clustered index?
    - Every thing is same for both the indexes i.e. it uses  B-TREE  for searching data.
    - However, the main difference is the way it stores physical data
    - In clustered when we reach the leaf nodes we are on the actual data. In non-clustered indexes, we get a pointer, which then points to the actual data.


24) Temp Table vs Table variable?
   - Temporary Tables are real tables so you can do things like CREATE INDEXes, 
     etc. If you have large amounts of data for which accessing by index will be faster then temporary tables are a good option.
   - Table variables cannot be involved in transactions, logging or locking. This makes @table faster then #temp. So table variable is faster then temporary table.
   - Table variables can have indexes by using PRIMARY KEY or UNIQUE constraints.
   - Table variables don't participate in transactions but temp table can.
   - Temp table stores in TempDB and Table variable store in Memory.

5) How to choose non-clustered index?
   - Identify Query Patterns:
     By analyzing which columns are frequently used for filtering, sorting & joining.
   - Choose Columns Based on WHERE Clauses which are frequently used
   - Avoid Over Indexing because it will cause additional overhead in terms of storage & maintenance.
   - Use Covering index which is basically index with c columns
      - This basically improves performence since required data is avialble in the index iteself 
      - There is no need to access main table.

4) What are the differences between local and global temporary tables?
   Local temparary tables:
   Local temporary tables  They are visible only to the connection that creates it, and are deleted when the connection is closed.
   Syntax: CREATE TABLE #<tablename>
   Global temporary tables:
   They are visible to all connections of SQLServer, and only destroyed when the last connection referencing the table is closed
   Syntax : CREATE TABLE ##<tablename>

17) Composite keys?
    - A composite key is a combination of two or more columns in a table that can be used to uniquely identify each row in the table 
    - when the columns are combined uniqueness is guaranteed, but when it taken individually it does not guarantee uniqueness.
    - Syntax:
      CREATE TABLE SAMPLE_TABLE  
     (COL1 integer,  
      COL2 nvarchar(30),  
      COL3 nvarchar(50),  
      PRIMARY KEY (COL1, COL2));  

20) How to improve query performence if its taking to much time to load? 
    - Define All Primary Keys and Foreign Key Relationships
    - Normalized database
    - Avoid * in SELECT statement. Give the name of columns which you require.
    - Create Clustered and Non-Clustered Indexes.
    - Use EXISTS instead of IN to check existence of data.
    - Choose appropriate Data Type. E.g. To store strings use varchar in place of text data type. Use text data type, 
      whenever you need to store large data (more than 8000 characters).
    - Avoid nchar and nvarchar if possible since both the data types takes just double memory as char and varchar.
    - Keep clustered index small since the fields used in clustered index may also used in non-clustered index.
    - Use joins instead of sub-queries.
    - Avoid Cursors since cursor are very slow in performance.
    - Use Table variable in place of Temp table.
    - Use Schema name before SQL objects name.
    - Use SET NOCOUNT ON and use TRY- CATCH to avoid deadlock condition
    - Avoid prefix  sp_  with user defined stored procedure name because SQL server first search the user defined procedure in the master database 
      and after that in the current session database.

21) What is CTE?
    - A common table expression (CTE) can be thought of as a temporary result set that you can reference within a SELECT, INSERT, UPDATE, or DELETE statement. 
    - It is similar to a derived table. Unlike a derived table, a CTE can be self-referencing, not stored as object and can be referenced multiple times in the same query.
    - It can be recursive and non-recursive.
    - It provides the significant advantage of being able to reference itself because  earlier version sql server, 
      a recursive query usually requires using temporary tables, cursors, and logic to control the flow of the recursive step.

23) Function
   - Basically it is a set of sql statements that accepts only input parameters, perform actions and return the result. 
   - Function can return only single value or a table. We can t use function to Insert, Update, Delete records in the database table(s).
   - Types: 
	1) System Defined Function
           - Scalar Function: Scalar functions operates on a single value and returns a single value. 
              - abs(-10.67) This returns absolute number of the given number means 10.67.
              - rand(10), lower('DOTNET'), upper('dotnet'), round(17.56719,3)	
           - Aggregate Function
              - Aggregate functions operates on a collection of values and returns a single value.
              - max(), min(), avg(), count()
        2) User Defined Function
           - Scalar Function: User defined scalar function also returns single value as a result of actions perform by function.
           - Inline Table-Valued Function
             - User defined inline table-valued function returns a table variable as a result of actions perform by function. 
             - The value of table variable should be derived from a single SELECT statement.
	   - Multi-Statement Table-Valued Function
             - User defined multi-statement table-valued function returns a table variable as a result of actions perform by function.
             - In this a table variable must be explicitly declared and defined whose value can be derived from a multiple sql statements.


25) Magic Tables in SQL Server
   - Magic Tables are invisible tables or virtual tables. You can see them only with the help Triggers in SQL Server.
   - You can also use the Magic tables with Non-Trigger activities using OUTPUT Clause in SQL Server 2005, 2008 & 2008 R2 versions.
   - Magic Tables are those tables which allow you to hold inserted, deleted and updated values during 
     insert, delete and update DML operations on a table in SQL Server.
   - These are the two Magic Tables: 
     1) Inserted
     2) Deleted
   - Whenever you Insert a record on that table, that record will be shown in the INSERTED Magic Table.
   - Whenever you Delete the record on that table, that record will be shown in the DELETED Magic Table Only. 
   - Whenever you Update the record on that table, that existing record will be shown in the DELETED Magic Table 
     and Updated new data will be shown in the INSERTED Magic Table.
